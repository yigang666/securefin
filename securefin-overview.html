App Design
server.js · logger.js · metrics.js

The application is intentionally small — a financial-transaction CRUD API — so the DevOps tooling takes centre stage. Four files make up the runtime: server.js wires routes and starts the HTTP server; logger.js exports a pino factory; metrics.js registers Prometheus counters and histograms; and server.test.js exercises every endpoint via supertest.

server.js
  • Express app factory (exported for supertest)
  • In-memory Map as transaction store
  • Input validation: amount positive, currency 3-letter ISO
  • Request logging middleware via logger
  • Metrics middleware increments counters on each request
  • app.listen() only called when not in test mode
logger.js
  • Exports a pino instance configured from LOG_LEVEL env var
  • Default level: info (prod/staging) or debug (dev)
  • Tests run with LOG_LEVEL=silent — no test output noise
  • JSON format in production; pretty-print via pino-pretty locally
  • Child logger per request with req.id correlation field
metrics.js
  • prom-client Registry with default Node.js metrics enabled
  • http_requests_total Counter — labels: method, route, status
  • http_request_duration_seconds Histogram — buckets: 0.005–10s
  • transactions_total Counter — labels: currency, type (credit/debit)
  • Exports registry and individual metrics for import in server.js
server.test.js
  • 8 test cases covering all endpoints
  • Uses supertest — no TCP port opened during tests
  • LOG_LEVEL=silent suppresses log output
  • Tests: POST valid, POST invalid amount, POST invalid currency, GET all, GET by ID, GET missing ID, GET /health, GET /metrics

API Endpoints
5 routes exposed by the Express app
POST/transactions

Creates a new transaction. Generates a UUID, stores in memory, returns 201 with full object.

// Request body
{ "amount": 99.50, "currency": "USD", "description": "Coffee" }

// 201 Response
{ "id": "f3a2...", "amount": 99.50, "currency": "USD",
  "description": "Coffee", "createdAt": "2026-03-17T10:00:00.000Z" }

// 400 — invalid amount or currency format
{ "error": "amount must be a positive number" }
GET/transactions

Returns all transactions as a JSON array. Empty array when no data. Always 200.

GET/transactions/:id

Returns a single transaction by UUID. Returns 404 with {"error":"not found"} for unknown IDs.

GET/health

Kubernetes liveness and readiness probe target. Returns {"status":"ok"} with 200. No dependencies checked.

GET/metrics

Prometheus text-format metrics exposition. Scraped by ServiceMonitor every 30s. Includes Node.js default metrics plus custom app metrics.

Dockerfile
Multi-stage Alpine, non-root, read-only filesystem
# ── Stage 1: deps ───────────────────────────────────────────
FROM node:20-alpine AS deps
WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production

# ── Stage 2: runtime ────────────────────────────────────────
FROM node:20-alpine AS runtime

RUN addgroup -g 1000 appgroup && \
    adduser -u 1000 -G appgroup -D appuser

WORKDIR /app
COPY --from=deps /app/node_modules ./node_modules
COPY --chown=appuser:appgroup . .

# /tmp needed because readOnlyRootFilesystem:true in K8s
VOLUME ["/tmp"]

USER 1000
EXPOSE 3000
HEALTHCHECK --interval=30s --timeout=3s --retries=3 \
  CMD wget -qO- http://localhost:3000/health || exit 1
CMD ["node", "server.js"]
Multi-stage build

Stage 1 installs only production dependencies with npm ci --only=production. Stage 2 copies the pruned node_modules — dev dependencies never enter the final image, reducing attack surface and image size.

Non-root execution

User appuser (UID 1000) created and set as USER before CMD. Kubernetes runAsNonRoot: true enforces this at admission — the pod will not start if the image tries to run as root.

Multi-architecture

GitHub Actions builds for linux/amd64 and linux/arm64 via docker/build-push-action with platforms matrix. A single manifest list is pushed to GHCR.

HEALTHCHECK

Docker-native health check uses wget (available in Alpine) hitting /health. Kubernetes liveness/readiness probes use the same endpoint but are defined separately in the Helm deployment template.

Security Hardening
Pod SecurityContext — every field explained
FieldValueEffect
allowPrivilegeEscalationfalsePrevents any child process gaining more privileges (no sudo, no setuid binaries)
readOnlyRootFilesystemtrueFilesystem is immutable at runtime — attackers cannot write tools or persist malware
capabilities.drop["ALL"]Every Linux kernel capability removed — no raw sockets, no bind to ports <1024, etc.
runAsNonRoottrueKubernetes admission controller rejects pod if image runs as UID 0
runAsUser / runAsGroup1000Explicit UID/GID — file permissions predictable and auditable
seccompProfile.typeRuntimeDefaultSyscall filtering — blocks ~300 system calls not needed by most apps
💡 /tmp emptyDir mount: because readOnlyRootFilesystem: true is set, Node.js (and npm, pino) need a writable directory for temporary files. An emptyDir volume is mounted at /tmp in the Deployment template. This is the only writable path in the container.

NetworkPolicy
Default-deny with minimal allow rules
# Default deny all ingress + egress
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
spec:
  podSelector: {}   # applies to all pods in namespace
  policyTypes: [Ingress, Egress]
  ingress:
  - from:
    - namespaceSelector:
        matchLabels:
          kubernetes.io/metadata.name: ingress-nginx
  egress:
  - ports:
    - protocol: UDP
      port: 53   # DNS resolution only
ℹ️ Only traffic from the ingress-nginx namespace is allowed in. All other pod-to-pod traffic, internet egress, and internal service mesh calls are blocked unless explicitly permitted. DNS (UDP 53) is the only egress allowed — required for Kubernetes service discovery.

ApplicationSet — 3-Environment Generator

apiVersion: argoproj.io/v1alpha1
kind: ApplicationSet
spec:
  generators:
  - list:
      elements:
      - env: dev      branch: dev      namespace: securefin-dev
      - env: staging  branch: staging  namespace: securefin-staging
      - env: prod     branch: main     namespace: securefin-prod
  template:
    spec:
      source:
        repoURL: https://github.com/org/securefin
        targetRevision: "{{branch}}"
        helm:
          valueFiles:
          - helm/values.yaml
          - helm/values.{{env}}.yaml   # conditional — only if file exists
      destination:
        namespace: "{{namespace}}"
      syncPolicy:
        automated:
          prune: true
          selfHeal: true   # auto-revert manual kubectl edits
